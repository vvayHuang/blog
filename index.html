2023六角學院網頁切版直播班心得

當初為什麼報名這堂課？

因為之前有學過，但是很久沒碰，有點生疏了，所以就報名這堂課，複習也順便增加自己對切版的熟練度

第一週~第八週的學習過程

上課的感覺很像又回到學生時期，差別在於線上課程很少會知道對方的長相，比較少有接觸的機會，這是比較可惜的

最大的收獲是？

認識與自己相同目標或興趣的朋友吧

最喜歡直播班的哪些活動？

禮拜五的直播，因為那是唯一有互動，可以看到“人”的部分

如果時光能倒流，會希望自己再次注意哪些細節？

希望自己可以在積極一點

對課程的建議或鼓勵？

希望直播可以再多一點，可能一個禮拜兩天，時間8-10，大約兩小時，抓重點講，因為有時候太長，聽到後面有點累，速度可以再快一點

沒有特別去和其他訓練營比較過，不過有感覺到六角學院算是蠻認真的一個組織，而且還是在高雄，身為也是高雄人的我特別欣慰，有點像在他鄉遇到一樣是南部或高雄上來的人，都會特別多聊兩句的感覺

最後也謝謝這段時間幫助過的朋友以及六角學院的朋友們

身為學長姐，分享些想入坑的新同學一些勉勵的話

學習是一種心態問題，即使今天花大錢學習，心態不對，你還是學不到東西

客製化 bootstrap5 scss

發表於 2022-08-10

如果要導入 bootstrap 所有元件與設定
只要把路徑改成檔案的位置，例如../node_modules/bootstrap/scss/functions

// Configuration
@import “functions”;
@import “variables”;
@import “maps”;
@import “mixins”;
@import “utilities”;

// Layout & components
@import “root”;
@import “reboot”;
@import “type”;
@import “images”;
@import “containers”;
@import “grid”;
@import “tables”;
@import “forms”;
@import “buttons”;
@import “transitions”;
@import “dropdown”;
@import “button-group”;
@import “nav”;
@import “navbar”;
@import “card”;
@import “accordion”;
@import “breadcrumb”;
@import “pagination”;
@import “badge”;
@import “alert”;
@import “progress”;
@import “list-group”;
@import “close”;
@import “toasts”;
@import “modal”;
@import “tooltip”;
@import “popover”;
@import “carousel”;
@import “spinners”;

但是因為不一定會用到所有的元件，可以把不需要的元件刪除

根據官方文件，可以這樣做
// Custom.scss
// Option B: Include parts of Bootstrap

// 1. Include functions first (so you can manipulate colors, SVGs, calc, etc)
@import “../node_modules/bootstrap/scss/functions”;

// 2. Include any default variable overrides here

// 3. Include remainder of required Bootstrap stylesheets
@import “../node_modules/bootstrap/scss/variables”;

// 4. Include any default map overrides here

// 5. Include remainder of required parts
@import “../node_modules/bootstrap/scss/maps”;
@import “../node_modules/bootstrap/scss/mixins”;
@import “../node_modules/bootstrap/scss/root”;

// 6. Optionally include any other parts as needed
@import “../node_modules/bootstrap/scss/utilities”;
@import “../node_modules/bootstrap/scss/reboot”;
@import “../node_modules/bootstrap/scss/type”;
@import “../node_modules/bootstrap/scss/images”;
@import “../node_modules/bootstrap/scss/containers”;
@import “../node_modules/bootstrap/scss/grid”;
@import “../node_modules/bootstrap/scss/helpers”;

// 7. Optionally include utilities API last to generate classes based on the Sass map in _utilities.scss
@import “../node_modules/bootstrap/scss/utilities/api”;

// 8. Add additional custom code here

一到七點基本上都是需要的，而且必須注意順序，如果不對編譯就會出現錯誤

客製化的部分就可以加在最後的地方

客製化元件在 scss 編寫的時候，要寫 before , after 這樣寫

&::before

要寫 hover, focus, active 這樣寫

&:hover

要寫在一起的話這樣寫

&:hover, &:focus

也可以操作狀態

例如 collapse 的屬性 [aria-expanded=”true”] 可以這樣寫

&[aria-expanded=”true”]

專案資料夾安裝 git 之後，基本設定會把 git 隱藏

如果要把它顯示有兩種方式：

  • 在電腦設定裡面把隱藏檔案打開顯示， mac 的快速件是 shift + command + .

  • 在 vscode 中打開設定（settings），右上角的地方點選 open setting(JSON)，在裡面輸入 files.exclude 指令，把 “**/.git” 改為 false，即可在左方列表看到 git 資料夾。

  • 電腦預設會把 git 資料夾隱藏，所以只要把隱藏打開就可以看到資料夾。macOS 打開隱藏檔的快速鍵是 shift + command + .

bootstrap scss 分類

發表於 2022-03-26

使用 bootstrap 會有客製化樣式的時候

客製化 scss 可以獨立出來一個資料夾並且把它分類，越細越好，這樣在修改往回尋找會比較好找，基本會分成元件（components）及排版通用類別（utilities）

把一個元件寫好，之後如果需要用同樣的就可以直接加上。通用類別則是一個東西可能需要的功能

也可以用同樣的方式，在 all.scss 的地方把字型（font）導入，這樣就不用加在 head 上

另外如果要用全域字型，在 variable.scss 中找到 font-family-sans-serif 在把字型名稱加到最前面即可

還有文字外框的 css 叫做
-webkit-text-stroke: 5px black;

粗細 顏色

hexo 樣式失效及設定網站站內連結

發表於 2022-03-24

如果樣式沒有出現，可能就是連結出現錯誤，仔細看程式碼就會看到

此時你需要進入 hexo 的 config.yml 將 relative_link 改成 true

重新編譯後會變成這個樣子

在檢查程式碼內的 css 及 js 連結，順利的話，完成後樣式就會出現

（由於每個主題樣式的安裝方式都不同，所以有可能會需要手動調整）

參考六角學院的 hexo 教學影片製作的朋友，應該會發現上傳至 github 把樣式搞定之後又會遇到一個問題，就是點選文章連結會找不到頁面，這個問題就是連結出錯了

例如你的網址是 vvayhuang.github.io/blog/，但是點選文章的時候會發現網址會變成 vvayhuang.github.io/年/月/日/文章名稱，中間的 blog 專案名稱不見了

此時就必須打開 hexo 的 config.yml ，在 url 的地方把你主頁面的網址貼上，意思就是未來文章就會在這個連結下打開，點選就不會找不到頁面

在 Hexo 安裝 GA

發表於 2022-03-23

Google Analytics 簡稱 GA，安裝 GA 可已知道來看你的網站的人士什麼時段、國家等等數據，所以一定要裝，安裝的方式很簡單

進入 GA 官網，點選左下角管理

選擇設定輔助程式

在網頁串流詳請頁面中找到新增網頁內代碼>全域網站代碼

接下來要把代碼放到網站上，根據 GA 的說明，是要把代碼放在網站的裡面，但因為一個網站有很多分頁，不可能一個一個慢慢加進去，所以一定更快的方式。

以我用的這個 Next 主題為例，代碼主要是放在主題資料架下的 layout.njk，路徑如下： themes>hexo-themes-next>layout>_partials>_layout.njk

每個主題的格式語言都不一樣，遇到時可再自行應變
